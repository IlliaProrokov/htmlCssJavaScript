## **Завдання 1: Персональний Вебсайт із Трьох Сторінок (Three-Page Personal Website)**

**Мета:** Створити зв'язаний багатосторінковий вебсайт із робочою навігацією.

**Файли, які потрібно створити:**

1.  `index.html` - Домашня сторінка (**Home page**).
2.  `about.html` - Сторінка "Про мене" (**About page**).
3.  `contact.html` - Сторінка "Контакти" (**Contact page**).

**Вимоги до КОЖНОЇ сторінки:**

  * Правильна **HTML5 structure** (структура HTML5 — це обов'язковий набір тегів, як-от `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>`).
  * Одинакове **navigation menu** (меню навігації) на всіх сторінках.
  * Узгоджені **header** (заголовок сторінки/шапка) і **footer** (нижній колонтитул/підвал).
  * Блок коментарів (**Comment block**) на початку файлу.

**Меню навігації (Navigation menu) (на ВСІХ сторінках):**

```html
<nav>
  <a href="index.html">Home</a> |
  <a href="about.html">About</a> |
  <a href="contact.html">Contact</a>
</nav>
```

  * Тег `<nav>` використовується для основного навігаційного контенту.
  * Тег `<a>` (anchor) створює посилання. Атрибут `href` вказує шлях до файлу.

**Вміст `index.html`:**

  * H1: "Welcome to My Website" (H1 — це найбільший заголовок).
  * 2-3 абзаци з представленням себе.
  * **Link** (посилання) на вашу сторінку `about.html`.
  * **Link** на зовнішній сайт (ваша школа, улюблений вебсайт тощо).

**Вміст `about.html`:**

  * H1: "About Me"
  * Секції з **IDs** (ідентифікаторами): `education`, `interests`, `goals`. *ID* — це унікальний ідентифікатор елемента, який дозволяє на нього посилатися.
  * **Navigation links** (навігаційні посилання) для переходу всередині сторінки до цих секцій.
  * Щонайменше 3 абзаци.

**Вміст `contact.html`:**

  * H1: "Get In Touch"
  * **Email link** (посилання на електронну пошту): `mailto:your@email.com`. *`mailto:`* — це протокол, що відкриває поштовий клієнт.
  * **Phone link** (посилання на телефон; вигаданий номер підходить): `tel:+14405551234`. *`tel:`* — це протокол, що дозволяє здійснити дзвінок на мобільному пристрої.
  * Посилання на соціальні мережі (можна вигадані для практики).

## **Завдання 2: Практика Якірних Посилань (Anchor Link Practice)**

**Мета:** Створити односторінковий сайт із внутрішньою навігацією.

**Створити:** `resources.html`

**Структура Вмісту (Content Structure):**

1.  Навігація у верхній частині з **anchor links** (якірними посиланнями). *Якірне посилання* — це посилання, яке перекидає користувача до певного місця (якоря) на тій самій сторінці.
2.  П'ять секцій з **IDs**.
3.  Посилання "Back to top" (Нагору) у кожній секції.

**Обов'язкові секції:**

  * HTML Resources (`#html`)
  * CSS Resources (`#css`)
  * JavaScript Resources (`#javascript`)
  * Tools (`#tools`)
  * Books (`#books`)

**Кожна секція повинна містити:**

  * Заголовок H2 з **ID**.
  * 2-3 абзаци вмісту.
  * Щонайменше 2 **external links** (зовнішні посилання) на реальні ресурси.
  * Якірне посилання "Back to top".

**Приклад:**

```html
<nav>
  <a href="#html">HTML</a> |
  <a href="#css">CSS</a> |
  <a href="#javascript">JavaScript</a> |
  <a href="#tools">Tools</a> |
  <a href="#books">Books</a>
</nav>

<section id="html">
  <h2>HTML Resources</h2>
  <p>Content...</p>
  <p><a href="https://www.w3schools.com" target="_blank">W3Schools</a></p>
  <p><a href="#top">Back to Top</a></p>
</section>
```

  * Зверніть увагу, що якірне посилання починається зі знаку `#`.
  * Атрибут `target="_blank"` відкриває посилання в новому вікні/вкладці.

## **Завдання 3: Бізнес-Вебсайт (Business Website)**

**Мета:** Створити професійну багатосторінкову структуру сайту.

**Файли, які потрібно створити:**

1.  `home.html`
2.  `services.html`
3.  `team.html`
4.  `contact-us.html`

**Бізнес:** Оберіть будь-який вигаданий бізнес (ресторан, магазин, технологічна компанія тощо).

**Вимоги:**

**`home.html`:**

  * Назва компанії та **tagline** (слоган).
  * Короткий опис.
  * Посилання на всі інші сторінки.
  * Телефонне посилання "Call us".
  * Поштове посилання "Email us".

**`services.html`:**

  * H1: "Our Services"
  * Три секції послуг із заголовками H2.
  * **Anchor links** для переходу до кожної послуги.
  * Посилання назад на домашню сторінку.

**`team.html`:**

  * H1: "Meet the Team"
  * Три біографії членів команди (**team member bios**).
  * **Email link** для кожної особи.
  * Посилання назад на домашню сторінку.

**`contact-us.html`:**

  * H1: "Contact Us"
  * **Email link**.
  * **Phone link**.
  * Адреса (використовуйте абзац із `<br />` для перенесення рядків). *`<br />`* створює примусовий розрив рядка.
  * Посилання, щоб відкрити Google Maps (зовнішнє, в новій вкладці).
  * Посилання назад на всі інші сторінки.

## **Поширені Помилки, Яких Слід Уникати (Common Mistakes to Avoid)**

1.  Забувати лапки (**quotes**) навколо значень `href`.
2.  Неправильно писати назви файлів (чутливість до регістру **case sensitive**\!).
3.  Забувати `#` для **anchor links**.
4.  Пропускати `mailto:` або `tel:` для пошти/телефону.
5.  Забувати `https://` для **external links**.
6.  Не закривати теги `</a>`.
7.  Розміщувати навігацію всередині тегів посилань.
