## **Завдання 1: Сторінка Рецепту з Форматуванням Тексту (Recipe Page with Text Formatting)**

**Objective (Мета):** Практика використання **text formatting tags** (`тегів форматування тексту`) та **HTML entities** (`HTML-сутностей`).

**Instructions (Інструкції):**

Створіть файл під назвою **`recipe.html`** з рецептом вашої улюбленої страви. Включіть:

### **Required Elements (Обов'язкові Елементи):**

1.  Правильна **HTML5 document structure** (`структура документа HTML5`) з **`DOCTYPE`** (`оголошення типу документа`).
2.  Кодування символів **`UTF-8`**.
3.  **Page title** (`Заголовок сторінки`): "**`[Назва Рецепту] - My Recipe Collection`**".
4.  Блок **comment** (`коментар`) на початку з вашою інформацією.

### **Required Content (Обов'язковий Вміст):**

1.  Головний **heading** (`заголовок`) (**`<h1>`**): Назва рецепту.
2.  Підзаголовок (**`<h2>`**): "**Ingredients**" (`Інгредієнти`).
3.  Перерахуйте інгредієнти (ми вивчимо правильні **lists** — *списки* — пізніше, поки що просто використовуйте **`paragraph`** — *абзаци*):
      - Використовуйте **`<strong>`** для **measurement amounts** (`кількості вимірювань`) (наприклад, **`<strong>2 cups</strong>`**).
      - Використовуйте **`<sub>`** для хімічних позначень або одиниць (наприклад, **`H<sub>2</sub>O`**).
4.  Підзаголовок (**`<h2>`**): "**Instructions**" (`Інструкції`).
5.  Пронумеруйте свої кроки, використовуючи **paragraphs** (`абзаци`) (Крок 1, Крок 2, тощо):
      - Використовуйте **`<em>`** для **emphasize** (`наголошення`) важливих **techniques** (`технік`).
      - Використовуйте **`<mark>`** для **highlight** (`виділення маркером`) основного **ingredient** (`інгредієнта`).
6.  Підзаголовок (**`<h2>`**): "**Chef's Notes**" (`Примітки Кухаря`).
7.  Додайте **paragraph** (`абзац`) з **cooking tips** (`порадами щодо приготування`).
      - Використовуйте **`<i>`** для іноземних слів (наприклад, *al dente*).
8.  Секція **footer** (`нижній колонтитул`) з:
      - Символом **copyright** (`авторського права`) (**`&copy;`**) та поточним роком.
      - Часом підготовки (**Prep time**) та часом приготування (**Cook time**), використовуючи **`<sup>`** для одиниць (наприклад, **`30 min<sup>*</sup>`**).

### **Example (Приклад):**

```html
<!-- Example of a single line of code you might use -->
<p>2. Gently fold in the cream. If you mix too fast, the texture will be <em>ruined</em>!</p>
<p>Total time: 45 min<sup>*</sup></p>
<p>&copy; 2024 My Recipe Collection</p>
```

## **Завдання 2: Структура Блогу з Семантичними Елементами (Blog Structure with Semantic Elements)**

**Objective (Мета):** Практика використання **semantic HTML** (`семантичного HTML`) для структурування сторінки.

**Instructions (Інструкції):**

Створіть файл під назвою **`blog.html`** із макетом для однієї **blog post** (`запису в блозі`).

### **Required Elements (Обов'язкові Елементи):**

1.  Правильна **HTML5 document structure** (`структура документа HTML5`).
2.  Використовуйте **CSS** (**Cascading Style Sheets** — *мова для стилізації*), щоб додати мінімальні стилі (наприклад, **`font-family`** *— сімейство шрифтів*, **`max-width`** *— максимальна ширина*), щоб вміст не був занадто великим.

### **Required Semantic Structure (Обов'язкова Семантична Структура):**

1.  **`<header>`**: Містить **`<h1>`** (Назва Блогу) та **`<nav>`** (Меню Навігації).
      - **`<nav>`** повинен містити посилання-заглушки (наприклад, `<a href="#">About</a>`).
2.  **`<main>`**: Містить **`<article>`** та **`<aside>`**.
      - **`<article>`** (*Автономний вміст, наприклад, сам запис у блозі*):
          - **`<h2>`** (Заголовок Запису).
          - **`<h3>`** (Дата та Автор).
          - **`<section>`** (*Тематичні розділи, наприклад, "Introduction" та "Conclusion"*): Містить принаймні два **paragraphs** (`абзаци`).
          - Використовуйте **`<hr>`** для відокремлення розділів.
      - **`<aside>`** (*Вміст, пов'язаний з основним, наприклад, бічна панель*):
          - **`<h3>`** (Назва бічної панелі, наприклад, "Related Topics").
          - Деякі **paragraphs** (`абзаци`) або **lists** (`списки`) пов'язаного вмісту.
3.  **`<footer>`**: Містить **copyright** (`авторське право`), контактну інформацію (використовуйте **`&copy;`** та **`<br>`**).

### **Example (Приклад):**

```html
<!-- Example structure you should use -->
<header>
    <h1>The Tech Blog</h1>
    <nav>
        <a href="#">Home</a>
        <a href="#">Articles</a>
    </nav>
</header>
<main>
    <article>
        <h2>The Future of Web Development</h2>
        <!-- ... content ... -->
    </article>
    <aside>
        <h3>Popular Posts</h3>
        <!-- ... related content ... -->
    </aside>
</main>
<footer>
    <!-- ... copyright info ... -->
</footer>
```

## **Поширені Помилки, Яких Слід Уникати (Common Mistakes to Avoid)**

1.  **Missing closing tags** (`Відсутність кінцевих тегів`) - Кожен **`<tag>`** потребує **`</tag>`**.
2.  **Mismatched tags** (`Невідповідність тегів`) - **`<strong></em>`** це неправильно\!
3.  **Using `<br />` for spacing** (`Використання <br /> для інтервалу`) - Використовуйте **CSS** замість цього (пізніше).
4.  **Skipping heading levels** (`Пропуск рівнів заголовків`) - Не переходьте від **`<h2>`** до **`<h5>`**.
5.  **Wrong semantic elements** (`Неправильні семантичні елементи`) - **`<footer>`** всередині **`<nav>`** не має сенсу.
6.  **Not using semantic HTML** (`Невикористання семантичного HTML`) - Використання **`<div>`** для всього (ми вивчимо **divs** пізніше).
